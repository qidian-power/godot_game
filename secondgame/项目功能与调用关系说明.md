# 挖掘项目功能与调用关系说明

本文档用于说明当前项目各部分的功能职责、数据分层，以及核心函数调用关系。

## 1. 项目结构总览

- 场景入口
  - `scenes/mine_stage.tscn`：主挖掘场景入口，挂载 `MineStageScene` 脚本。
  - `scenes/ui/dig_cell.tscn`：单个可点击格子场景，挂载 `DigCell` 脚本。
- 逻辑脚本
  - `scripts/scenes/mine_stage_scene.gd`：场景编排/UI/交互中枢。
  - `scripts/systems/mine_stage_system.gd`：关卡状态与结算规则。
  - `scripts/systems/grid_system.gd`：网格状态存储与挖掘结果计算。
  - `scripts/systems/mineral_spawn_system.gd`：矿物按权重与尺寸生成。
  - `scripts/ui/dig_cell.gd`：格子点击与绘制逻辑。
- 配置脚本（Resource）
  - `scripts/config/mine_level_config.gd`：关卡参数定义。
  - `scripts/config/mineral_config.gd`：矿物参数定义。
  - `scripts/config/spawn_rule.gd`：生成规则参数定义。
  - `scripts/config/upgrade_config.gd`：升级参数定义。
  - `scripts/config/mine_content_config.gd`：总配置容器（关卡/矿物/升级）。
- 默认兜底配置
  - `scripts/data/default_configs.gd`：代码构建的默认数据（当资源缺失时回退）。
- 配置资源（.tres）
  - `resources/mine_content_default.tres`：总配置入口。
  - `resources/levels/*.tres`、`resources/minerals/*.tres`、`resources/upgrades/*.tres`、`resources/rules/*.tres`：具体可编辑数据。

## 2. 分层职责（数据配置 vs 逻辑执行）

### 2.1 配置层（只描述数据）

- `MineLevelConfig`：网格大小、初始能量、每次消耗、矿物池 ID、生成数量、结束条件、特殊奖励概率等。
- `MineralConfig`：矿物尺寸（`width/height`）、价值、权重、血量。
- `SpawnRule`：不重叠/边界检查/最小间距等生成约束。
- `UpgradeConfig`：升级类型与价格、数值。
- `MineContentConfig`：把多关卡、多矿物、多升级集中成一个可挂载资源。

### 2.2 逻辑层（读取配置并执行规则）

- `MineStageScene`：读取配置、初始化系统、接收点击、刷新 UI、处理按钮与结算弹窗。
- `MineralSpawnSystem`：依据配置生成矿物占位。
- `GridSystem`：管理每格 hp/是否挖空，统计矿物块是否完全挖出。
- `MineStageSystem`：管理能量、金币、特殊奖励、结束判定。
- `DigCell`：单格可视化与点击信号。

## 3. 关键文件职责与核心函数

## 3.1 `scripts/scenes/mine_stage_scene.gd`

- 核心职责
  - 构建整个场景 UI（背景、信息区、调试区、升级区、挖掘区、结算面板）。
  - 启动流程与重开流程。
  - 连接 `DigCell` 点击信号并驱动挖掘。
  - 从资源或默认配置装配关卡、矿物、升级。
- 主要函数
  - `_ready()`：入口，调用 `_build_ui()` 与 `_start_stage()`。
  - `_build_ui()`：创建所有 UI 节点并绑定按钮事件。
  - `_start_stage()`：读取配置 -> 生成矿物 -> 建格子 -> 初始化系统 -> 刷新 UI。
  - `_rebuild_grid_nodes()`：实例化 `DigCell` 场景并绑定 `cell_clicked`。
  - `_on_cell_clicked(cell_id)`：处理挖掘点击与提示文案。
  - `_refresh_ui()`：刷新能量、矿物进度、金币、关卡信息与升级按钮文案。
  - `_resolve_content_data()` / `_resolve_level_config()` / `_build_mineral_map()` / `_resolve_upgrades()`：配置读取入口。
  - `_on_add_gold_pressed()` / `_on_restart_pressed()` / `_on_apply_level_pressed()`：调试与关卡切换。

## 3.2 `scripts/systems/mine_stage_system.gd`

- 核心职责
  - 维护局内状态：能量、力量、金币、总收益、是否结束。
  - 按配置执行挖掘消耗、收益计算、特殊奖励、结束判定。
- 主要函数
  - `setup(level_config, grid_system)`：注入依赖并 `reset_stage()`。
  - `dig_cell(cell_id)`：单次挖掘主逻辑，返回结果字典。
  - `check_end_condition()`：根据关卡结束类型判断是否结束。
  - `reset_stage()`：重置局内核心状态（含金币清零）。

## 3.3 `scripts/systems/grid_system.gd`

- 核心职责
  - 构建网格数据结构（每格 hp、是否矿物格、所属矿物块）。
  - 处理“格子受伤 -> 挖空 -> 是否完整挖出矿物块”的状态迁移。
- 主要函数
  - `build_grid(level_config, placements)`：根据生成结果初始化全格状态。
  - `can_dig(cell_id)`：格子是否可挖。
  - `dig_cell(cell_id, dig_power)`：更新 hp 和挖掘状态，返回挖掘结果。
  - `all_mined()`：是否已挖出全部矿物块（以 deposit 为单位）。

## 3.4 `scripts/systems/mineral_spawn_system.gd`

- 核心职责
  - 按权重随机矿物。
  - 按矿物尺寸在网格中尝试放置。
  - 处理不重叠和最小间距校验。
- 主要函数
  - `spawn_minerals(level_config, minerals)`：生成矿物占位结果 `placements`。
  - `_pick_by_weight(minerals, total_weight)`：加权随机矿物。
  - `_pick_cells_for_mineral(...)`：找一组可放置格子。
  - `_check_min_distance(...)`：最小间距约束。

## 3.5 `scripts/ui/dig_cell.gd`

- 核心职责
  - 单格点击输入处理。
  - 六边形格子绘制（填充/边框/血条/状态颜色）。
- 主要函数
  - `setup(cell_id, hp, mineral_label)`：初始化单格数据。
  - `_gui_input(event)`：左键点击发射 `cell_clicked` 信号。
  - `apply_damage(amount)`：扣 hp 并标记挖空。
  - `set_locked()/set_unlocked()`：切换可操作状态。
  - `_draw()` / `_get_fill_color()`：可视化逻辑。

## 4. 核心流程调用链

### 4.1 场景启动流程

1. `MineStageScene._ready()`
2. `MineStageScene._build_ui()`
3. `MineStageScene._start_stage()`
4. `MineStageScene._build_mineral_map()` / `_resolve_level_config()` / `_resolve_upgrades()`
5. `MineralSpawnSystem.spawn_minerals(level_config, available_minerals)`
6. `GridSystem.build_grid(level_config, placements)`
7. `MineStageSystem.setup(level_config, grid_system)`
8. `MineStageScene._rebuild_grid_nodes()`
9. `MineStageScene._refresh_ui()`

### 4.2 点击挖掘流程

1. `DigCell._gui_input()` 发射 `cell_clicked(cell_id)`
2. `MineStageScene._on_cell_clicked(cell_id)` 接收信号
3. `MineStageScene` 调用 `MineStageSystem.dig_cell(cell_id)`
4. `MineStageSystem.dig_cell()`
   - 调用 `GridSystem.can_dig(cell_id)`
   - 调用 `GridSystem.dig_cell(cell_id, dig_power)`
   - 调用 `MineStageSystem.check_end_condition()`
5. `MineStageScene._on_cell_clicked()` 根据结果调用 `DigCell.apply_damage()` 并更新提示
6. `MineStageScene._refresh_ui()`
7. 若结束：`MineStageScene._finish_stage(reason)` -> 锁定全部格子 + 显示结算

### 4.3 重开与切关

- 重开：`_on_restart_pressed()` -> `_start_stage()` -> 全流程重置
- 切关：`_on_apply_level_pressed()` -> 更新 `level_index` -> `_start_stage()`

## 5. 关键跨文件函数调用清单

- `scripts/scenes/mine_stage_scene.gd`
  - 调用 `scripts/systems/mineral_spawn_system.gd` 的 `spawn_minerals()`
  - 调用 `scripts/systems/grid_system.gd` 的 `build_grid()`
  - 调用 `scripts/systems/mine_stage_system.gd` 的 `setup()`、`dig_cell()`
  - 调用 `scripts/ui/dig_cell.gd` 的 `setup()`、`apply_damage()`、`set_locked()`
- `scripts/systems/mine_stage_system.gd`
  - 调用 `scripts/systems/grid_system.gd` 的 `can_dig()`、`dig_cell()`、`all_mined()`
- `scripts/systems/mineral_spawn_system.gd`
  - 读取 `scripts/config/mine_level_config.gd` 与 `scripts/config/mineral_config.gd` 数据
  - 读取 `scripts/config/spawn_rule.gd` 中约束字段
- `scripts/scenes/mine_stage_scene.gd`
  - 读取 `scripts/config/mine_content_config.gd` 聚合配置
  - 兜底调用 `scripts/data/default_configs.gd` 的 `build_minerals()`、`build_level()`、`build_upgrades()`

## 6. 数据资源加载优先级

- 关卡：`content_data.levels[level_index]` > `level_data` > `resources/levels/mine_level_1.tres` > `DefaultConfigs.build_level()`
- 矿物：`mineral_data` > `content_data.minerals` > `resources/minerals/*.tres` > `DefaultConfigs.build_minerals()`
- 升级：`upgrade_data` > `content_data.upgrades` > `resources/upgrades/*.tres` > `DefaultConfigs.build_upgrades()`

## 7. 调试与工具职责

- 调试加钱按钮：`MineStageScene._on_add_gold_pressed()`，直接对 `MineStageSystem.gold += 1000`。
- 重置本局按钮：`MineStageScene._on_restart_pressed()`，重新执行 `_start_stage()`。
- 切换关卡按钮：`MineStageScene._on_apply_level_pressed()`，切换 `level_index` 并重开。

## 8. 当前可扩展点（建议）

- 增加 `DIG_RANGE` 升级实际逻辑（目前枚举存在，尚未实现范围挖掘）。
- `SpawnRule.no_overlap` / `border_check` 当前主要通过放置逻辑隐式满足，可扩展为显式分支开关。
- `background_sprite` 可接入到 `MineStageScene` 动态背景读取。
